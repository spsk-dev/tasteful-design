**Create `skills/design-review/references/generation.md`:**

This file is read by `/design-improve` during page generation (Phase A build). It is NOT used by `/design-review` specialists during evaluation. The file provides positive aesthetics guidance to steer Claude away from generic AI output when building pages.

Structure the file with these sections:

```markdown
# Generation Aesthetics Reference

This reference guides page generation in `/design-improve`. It is adapted from Anthropic's DISTILLED_AESTHETICS_PROMPT with SpSk-specific extensions.

## Core Principle

[Paraphrase the opening: avoid generic "on distribution" output, make creative distinctive frontends that surprise and delight.]

## Typography

[Anthropic's typography guidance + SpSk extensions:]
- Choose distinctive fonts, not generic defaults (Inter, Roboto, Arial, system fonts)
- Use at most 2 font families + optional mono
- Good starting points by context:
  - Startup/SaaS: Clash Display, Satoshi, Cabinet Grotesk, Plus Jakarta Sans
  - Editorial/Content: Instrument Serif, Newsreader, Fraunces, Literata
  - Technical/Docs: IBM Plex, Source Sans 3, Fira Code
  - Distinctive: Bricolage Grotesque, Obviously, DM Sans
- Avoid convergence on common AI choices (Space Grotesk is becoming overused outside code contexts)
- Playfair Display: acceptable for genuine editorial (long-form articles, magazine layouts), NOT for SaaS/landing hero text

## Color and Theme

[Anthropic's color guidance + SpSk extensions:]
- Commit to a cohesive aesthetic with CSS variables
- Dominant color with sharp accents outperforms evenly-distributed palettes (60/30/10 rule)
- Draw from IDE themes, cultural aesthetics for inspiration
- Avoid: purple gradients on white backgrounds, dark+gold, synthwave, gray-everything
- Vary between light and dark themes across projects

## Motion

[Anthropic's motion guidance + SpSk extensions:]
- One well-orchestrated page load with staggered reveals (animation-delay) creates more delight than scattered micro-interactions
- Prioritize CSS-only solutions for HTML pages
- Use proper easing (ease-out for entrances, ease-in for exits), not linear
- Duration: 150-300ms for micro-interactions, 300-500ms for layout transitions
- Always include prefers-reduced-motion support

## Backgrounds

[NEW -- Anthropic adds this, SpSk did not previously have generation guidance for backgrounds:]
- Create atmosphere and depth rather than defaulting to solid colors
- Layer CSS gradients for richness
- Use geometric patterns or contextual effects that match the overall aesthetic
- Subtle noise textures, gradient meshes, or radial gradients add visual interest

## Variation Enforcement

[NEW -- from Anthropic's guidance:]
- Vary between light and dark themes across generations
- Do not converge on the same font choices across projects
- Each page should have a distinct visual identity, not interchangeable with any other
- If you notice you are reaching for the same patterns, deliberately choose something different

## Anti-Patterns (Do Not Generate)

- Overused font families: Inter, Roboto, Arial, system fonts
- Cliched color schemes: purple gradients, dark+gold, synthwave, gray-everything
- Predictable layouts: three-column icon grid, badge-hero-features-steps-testimonials-CTA skeleton
- Emoji as icons: use SVG icon libraries (Lucide, Phosphor, Heroicons)
- Cookie-cutter design: no context-specific character
```

Keep the file under 80 lines to limit token impact.
