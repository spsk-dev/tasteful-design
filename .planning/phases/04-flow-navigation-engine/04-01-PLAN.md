**skills/design-review/references/flow.md** (per D-75):
A reference document the agent loads during flow audit. Must cover:
1. Screen detection heuristics — DOM stability via MutationObserver (800ms quiet = stable), font readiness check via document.fonts.ready, never use networkidle
2. Flow intent mapping patterns — how to match user's flow description to visible CTAs (buttons, links, form submits), prioritize primary CTAs (larger, higher contrast, prominent position)
3. SPA navigation patterns — pushState/replaceState detection, in-page transitions (tabs, wizards, modals), URL change vs content-only change
4. Dead end detection — no matching CTAs, error pages (4xx/5xx), "page not found" content, infinite redirect loops
5. Success state detection — confirmation messages ("Thank you", "Success", "Complete"), order confirmations, redirect to dashboard/home after flow
6. Cookie/popup handling — dismiss cookie consent banners, handle promotional modals, distinguish flow-relevant modals from interruptions
7. Screenshot timing — after DOM stable + fonts ready, before capturing. Include the JS snippets from RESEARCH.md (DOM stability MutationObserver and font readiness check)
8. Slug generation — h1/h2 heading text > URL path segment > "screen-N" fallback, slugify rules

Write this as agent-consumable knowledge (the agent reads it during flow execution). Keep it practical with inline code snippets, not abstract theory.
