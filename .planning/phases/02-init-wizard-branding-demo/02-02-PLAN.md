Create `commands/design-init.md` -- the init wizard command file.

**Frontmatter** (per D-16, matching existing command pattern): Use standard YAML frontmatter delimiters. Set name to `design-init`. Set description to: "Interactive setup wizard for design tokens. Creates .design/ directory with tokens.json (colors, fonts, mode) and config.json (page type, vibe preset). Run once per project. Takes under 2 minutes." Set allowed-tools to: Read, Write, Bash(mkdir *)

CRITICAL: Do NOT list AskUserQuestion in `allowed-tools` -- it is always available by default and listing it can cause silent empty responses (Pitfall 1 from research).

**Command body -- write as Claude instructions (this is a prompt, not code):**

The command instructs Claude to run a 5-question interactive wizard:

**Q1: Page Type (per D-17 / INIT-02)**
Use AskUserQuestion to ask the user to select their page type. Present these options:
1. `landing` -- Landing pages, marketing sites, product pages
2. `dashboard` -- Data dashboards, analytics, admin panels
3. `admin` -- Back-office tools, CMS, internal tools
4. `docs` -- Documentation sites, knowledge bases, wikis
5. `portfolio` -- Personal sites, showcases, creative portfolios

Default: `landing`

**Q2: Vibe Preset (per D-17 / INIT-03)**
Use AskUserQuestion to present vibe presets. Show human-friendly labels with descriptions, but map internally to the `config/style-presets.json` keys (per Pitfall 4 from research):

Mapping (corrected per review -- semantic alignment with preset descriptions):
- "Corporate" (Clean, functional, data-dense) -> preset key `serious-dashboard`
- "Editorial" (Typography-driven, content-first, quiet elegance) -> preset key `minimal-editorial`
- "Playful" (Playful, warm, personality-forward) -> preset key `fun-lighthearted`
- "Bold" (Bold, conversion-focused, energetic) -> preset key `startup-landing`
- "Minimal" (Motion-forward, cinematic, immersive) -> preset key `animation-heavy`

Show the preset `description` field from `config/style-presets.json` as context for each option. Read the file at `@${CLAUDE_PLUGIN_ROOT}/config/style-presets.json` to get descriptions.

Default: based on page type (landing -> `startup-landing`, dashboard -> `serious-dashboard`, admin -> `serious-dashboard`, docs -> `minimal-editorial`, portfolio -> `animation-heavy`)

**Q3: Mode (per D-17 / INIT-04)**
Use AskUserQuestion: `light`, `dark`, or `both`. Default: `dark`

**Q4: Brand Colors (per D-17 / INIT-05 / D-19)**
Use AskUserQuestion: Ask for brand colors as hex values (e.g., "primary: #0F172A, accent: #3B82F6") or type "skip" for palette suggestions.

If user provides colors: parse hex values and use them for tokens.json colors.

If user types "skip" or similar: Read `@${CLAUDE_PLUGIN_ROOT}/config/palettes.json`, look up the page type from Q1, present the 3 named palettes with their colors. Use AskUserQuestion to let user pick one of the 3 palettes (show palette name and all 6 color values). Use the selected palette colors for tokens.json.

**Q5: Font Preference (per D-17 / INIT-06 / D-20)**
Use AskUserQuestion: Ask for font preference (e.g., "Geist", "Inter", "Instrument Serif") or type "skip" for a suggestion.

If user types "skip" or similar: Look up the selected vibe preset from Q2 in `config/style-presets.json` and extract the `typography` field. Suggest the primary font mentioned there. For example, `startup-landing` -> "Geist", `minimal-editorial` -> "Instrument Serif".

Set mono font to "Geist Mono" by default (consistent with SpSk branding).

**Output (per D-18 / INIT-07):**

After all 5 questions, create the `.design/` directory in the user's CWD (NOT in `${CLAUDE_PLUGIN_ROOT}` -- this is a project-level directory):

1. Run `mkdir -p .design`
2. Write `.design/tokens.json` with: colors object (primary, secondary, accent, background, foreground, muted from Q4), fonts object (primary from Q5, secondary derived from vibe, mono as "Geist Mono"), mode from Q3
3. Write `.design/config.json` with: page_type from Q1, vibe_preset as the preset KEY from Q2 (not human label), initialized as ISO-8601 timestamp

**Completion message:**
After writing files, confirm with a summary showing what was configured. Use branded output (load `@${CLAUDE_PLUGIN_ROOT}/shared/output.md` for format reference). Show the created files and their key values.
