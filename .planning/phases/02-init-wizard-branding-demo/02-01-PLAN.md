Create `shared/output.md` -- the branding reference file loaded by all commands via `@${CLAUDE_PLUGIN_ROOT}/shared/output.md`. This is a markdown reference (instructions for Claude), not executable code. Include these sections:

**Signature Line (per D-26):**
```
 SpSk  {command}  v{version}  ───  {specialist_count} specialists  ·  tier {tier}
```
- Read version from `${CLAUDE_PLUGIN_ROOT}/VERSION`
- specialist_count: 8 for full mode, 4 for quick mode
- tier: from environment detection (1=full, 2=no Gemini, 3=code-only)

**Score Bar (per D-28 -- /10 display scale):**
- 10 blocks total, filled char: `█`, empty char: `░`
- Internal scores are on 1.0-4.0 scale. Convert to /10 display: `display = internal * 2.5`
- Mapping: `round(score * 2.5 / 10 * 10)` filled blocks (equivalently: `round(score * 2.5)`)
- Display format: `████████░░ 8.0/10`
- Include examples: 4.0 internal = `██████████ 10.0/10`, 3.2 internal = `████████░░ 8.0/10`, 2.0 internal = `█████░░░░░ 5.0/10`, 1.0 internal = `███░░░░░░░ 2.5/10`
- IMPORTANT: Always display as /10 to users. Internal 1.0-4.0 scale is multiplied by 2.5 for display.

**Symbol Vocabulary (per D-29):**
- Pass/complete: `✓`
- Fail/missing: `✗`
- In progress: `◆`
- Pending: `○`
- Auto-approved: `⚡`
- Warning: `⚠`

**Box Drawing (per D-30 -- SINGLE-line borders, NOT double):**
```
┌─────────────────────────────────────────────────────────────┐
│  Content here                                               │
└─────────────────────────────────────────────────────────────┘
```
Use `┌ ─ ┐ │ └ ┘` characters. Do NOT use double-line `╔ ═ ╗ ║ ╚ ╝` (that is GSD style, not SpSk).

**Section Headers:**
Use single-line box for section headers within review output:
```
┌─ TYPOGRAPHY ─────────────────────────────────────────────────┐
│  ████████░░ 8.0/10                                           │
│  ✓ Font hierarchy clear                                      │
│  ✗ Body text too small (14px, recommend 16-18px)             │
└──────────────────────────────────────────────────────────────┘
```

**Footer (per D-31):**
Every output ends with a subtle footer line:
```
github.com/felipemachado/spsk
```

**Overall Structure Instruction:**
Tell Claude to use this format for ALL SpSk command output. Start with signature line, use boxes for each section, end with footer. Keep output compact and professional -- NOT big ASCII art (per project CLAUDE.md).

- Each page type's 3 palettes should feel genuinely different (not just hue shifts)

Example palette names by page type:
- landing: bold/energetic names (e.g., "Neon Launchpad", "Warm Catalyst", "Arctic Edge")
- dashboard: professional/functional names (e.g., "Midnight Operations", "Slate Control", "Carbon Grid")
